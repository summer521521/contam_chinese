EnergyPlus 输入文件

这些输入文件用于耦合过程，以通过 ContamFMU.dll 协调 EnergyPlus 和 CONTAM 之间的联合仿真期间的数据传输。这些文件可以使用 CONTAM3DExportTool 创建，但其内容在此处提供。

< 项目 > .vef

这个 变量交换文件 ContamFMU.dll 使用它来在协同仿真运行开始时协调数据交换信息。该文件将包含协同仿真期间要交换的变量列表。变量的数量和类型将取决于 CONTAM 项目文件的内容。

下面简要描述各个变量。单位在括号 [ ] 内表示。变量的类型在括号中。

(I) = 整数值，

(R) = 实数/小数值，

（字符） = 单个字符

（字符串） = 字符串

文件格式：

第一行是文件头，指示文件类型、版本信息和日志记录级别。
日志级别 可以设置在 0 到 3 之间，以向 ContamFMU.log 文件提供递增级别的数据交换信息：

变量交换文件 1.0 fmi log_levl

第二行提供了正在交换的变量的数量：

n 变量 (I)

每行将提供与联合仿真期间要交换的单个变量相关的信息：  

输入/输出指示灯 //（字符）“I”用于 ContamX 的输入或“O”用于 ContamX 的输出。

模型描述.xml

EnergyPlus 联合仿真主算法使用该文件来协调数据交换。其内容的一般要求由FMI联合仿真规范定义[ 莫迪萨 2010a , 莫迪萨尔 2010b ]。唯一的变量名称将用作 功能模型单元导入 EnergyPlus 中的对象。

VEF 文件中变量出现的顺序必须与 XML 文件中的值引用编号匹配。变量的名称不需要匹配。
