EnergyPlus-CONTAM 耦合过程

以下概述描述了将 CONTAM 项目与 EnergyPlus 耦合的过程。它涵盖了基本方法，但并未涵盖所有可能的耦合场景所需的所有细节。 该过程利用了 CONTAM3DExport 用于从 CONTAM PRJ 文件生成 EnergyPlus IDF 文件的工具。

开发耦合项目并不是绝对需要遵循此过程。人们可以开发一个 CONTAM 项目，该项目具有与现有 EnergyPlus IDF 相对应的热房间，使用 CONTAM3DExport 工具生成 IDF 文件，并将生成的 IDF 文件的部分内容合并到 EnergyPlus IDF 文件中。但是，如果不执行此过程进行耦合，则需要小心提供地下以包含关联的 CONTAM 项目的各个流动路径。此外，还需要协调生成的 EnergyPlus IDF 文件的空气回路和节点。

使用 ContamW 创建 CONTAM 项目

创建与 EnergyPlus 结合使用的 CONTAM 项目时，请遵循以下准则。表中的 与 EnergyPlus 合作 部分显示在 EnergyPlus 模型中创建的对象和相应的 CONTAM 实体。

几何

利用 伪几何 ContamW 模式，以便 CONTAM3DExport 生成 EnergyPlus 模型所需的几何形状。对于 CONTAM 模型中的每个房间，将在 EnergyPlus 模型中创建一个热房间。

气流路径

气流路径将被分组以形成 EnergyPlus 模型中的房间间混合和渗透气流速率。

窗户

要通过 EnergyPlus 创建用于热建模的窗口，请在 CONTAM 项目中定义气流路径，该项目使用名称包含字符串“wind”（不区分大小写）的双向气流元件。将气流路径图标放置在您希望窗户中心所在的墙壁上。通过气流路径参考标高定义窗口底部的标高，并通过气流元件属性定义窗口的高度和宽度。如果您不希望在耦合模拟期间有任何气流通过窗口，请将路径乘数设置为零。导出时，将根据这些属性在 IDF 文件中创建窗口。您必须确保窗户完全包含在墙内并且不会相互重叠。

简单的空气处理系统

将在 EnergyPlus 模型中为 CONTAM 模型中的每个简单空气处理系统 (AHS) 创建一个空气回路，但下文所述的纯排气和热回收通风机 (HRV) 系统除外。为了遵守 EnergyPlus 空气环路范例，空气环路服务的每个热房间都应具有相应的送风和回风终端，由 CONTAM 建筑模型中关联的“正常”简单 AHS 提供服务。一个房间只能由一个空气环路提供服务，因此 CONTAM 中的一个房间不应有超过一个正常的 AHS 提供服务。然而，一个房间可以由每种类型的 AHS 之一提供服务：正常、仅排气和 HRV；如下所述。

CONTAM 中的简单 AHS 可以对应于 EnergyPlus 中的仅排气系统和能量回收系统。简单的 AHS 的名称用于触发 CONTAM3DExport 创建这些系统以及系统特定的供应和返回终端要求。仅排气系统名称必须包含“exh”（不区分大小写），并且不应有任何与之关联的送风终端。能量回收系统名称应包含“hrv”（不区分大小写）。 CONTAM3DExport 将根据这些要求对 PRJ 文件执行检查。仅排气和 HRV 简单 AHS 在 EnergyPlus 中没有与其关联的相应空气回路。相反，HRV 系统将生成 暖通空调区：能量回收通风机 以及 IDF 文件中的关联对象，仅排气系统将生成 风扇：房间排气 以及 IDF 文件中的关联对象。

Contam3D导出 提供了一个选项，可以提供与简单 AHS 相关的空气回路，该简单 AHS 仅包含一个风扇以将空气移动通过回路，或者将两种类型的热调节系统之一与空气回路合并。用户可以选择包括 AirLoopHVAC：UnitaryHeatCool or AirLoopHVAC：一体式热泵：AirToAir 每个空气循环。这将提供必要的组件来调节与每个空气回路相关的热房间。可能需要对导出的 IDF 进行修改才能完成加热/冷却系统的定义。修改包括设置单一系统的控制区或恒温器位置、设置 房间渗透：设计流量 以及系统大小调整的关联房间列表，以及所需系统性能所需的导出系统的任何其他属性。

管道

与 EnergyPlus 耦合时目前不支持管道。 CONTAM3DExport 不会对此执行检查。

控制

控件可用于在 CONTAM 和 EnergyPlus 之间传递数据。这允许在两个程序之间共享几乎任何信息，并且由用户定义在 CONTAM 控制网络和 EnergyPlus 能源管理系统 (EMS) 中使用哪些数据以及如何使用这些数据。 CONTAM3DExport 将提供如下定义的数据传输发生的基础。

在 CONTAM 中 常数 (or Set ) 具有名称的类型控件用于接收来自 EnergyPlus 的输入控制信号。 斯普利特 (or 通行证 ) 具有名称的类型控件用于将输出信号从 CONTAM 发送到 EnergyPlus。在使用具有控件的项目执行模拟之前，请确保仅运行 CONTAM 模拟，或者至少通过模拟运行完整的构建检查 > 运行 Building Check 菜单项以确保控件按顺序排列。将任何新控件添加到项目后，必须遵循此过程，并且还必须在控件排序后保存项目。

运行 CONTAM3DExport

使用以下命令创建 EnergyPlus IDF 文件 CONTAM3DExport

  1. Run CONTAM3DExport 并选择 PRJ 文件以转换为 EnergyPlus。
  2. 选择包含以下内容的现有 IDF 文件 材质 and 建筑工程 您想要合并到新 IDF 文件中的对象。

    3. 注意： CONTAM3DExport 将为地板、天花板和内墙创建相邻表面，因此，如果在两侧使用相同的结构，则对于这些类型的表面或对称层具有反向层的结构有利于耦合过程。 CONTAM3DExporter 创造 建筑表面：详细 对这些表面使用表面类型外部边界条件的对象。

  3. 选择要与每个表面类别关联的构造。
  4. 选择 与 CONTAM 相结合的建筑    导出选项。
  5. 选择所需的空气循环选项。此选项将应用于 CONTAM 项目中的每个简单空气处理系统 (AHS)，除非定义为排气或热回收 AHS。
    1. 仅限粉丝 - 创建 空气循环暖通空调 有风扇但没有加热或冷却组件。
    2. 一体式热冷 - 包括 AirLoopHVAC：UnitaryHeatCool 空间调节系统。
    3. 空气对空气热泵 - 包括 一体式热泵：AirToAir 空间调节系统。
  6. 单击保存 EnergyPlus IDF 按钮。导出完成后将会提示您。

文件创建者 CONTAM3DExport

  • IDF文件 - 包含几何结构、空气回路、排气扇和 FMU 相关对象。
  • CONTAMFMU.fmu - 一个压缩的 zip 文件，将由 EnergyPlus 解压并在耦合的 EnergyPlus/CONTAM 模拟期间使用。它包含以下文件：
    • 模型描述.xml - EnergyPlus FMI 联合仿真主算法使用它来获取数据交换参数。
    • 污染源 - V 可变的 E 交换 F ContamX 使用该文件来获取数据交换参数。
    • 污染项目 - CONTAM 项目文件，用于生成相应的 EnergyPlus IDF 文件并由 ContamX 在联合仿真期间使用。
    • ContamFMU.dll - EnergyPlus主算法利用动态链接库来实现基于FMI的程序控制以及与ContamX的数据交换。
    • contamx3.exe - CONTAM 仿真引擎由 EnergyPlus FMI 联合仿真主算法通过 ContamFMU.dll 控制。

修改IDF文件

生成的IDF文件 CONTAM3DExport 包含如上所述的建筑几何形状、表面材料和结构、空气回路、排气扇和外部接口对象。该文件还将包括与产生或消除能量相关的 HVAC 组件，例如加热/冷却盘管。虽然耦合仿真应按导出方式运行，但它不太可能按预期执行。它不包括内部增益、接地耦合或与水相关的项目，并且 HVAC 系统可能不会包含所有所需的组件、所创建组件的所需尺寸（例如线圈和风扇），并且应根据需要修改恒温设定点。

现在可以使用 IDF 编辑器或其他基于文本的编辑器来编辑 IDF 文件，以添加或修改所需的组件。还可以使用 OpenStudio 或其关联的 Sketchup 插件等工具查看和编辑 IDF 文件 古列梅蒂 2011 , 国家可再生能源实验室2015 ]。例如，可以将 IDF 文件导入 OpenStudio，并且可以使用几何查看器来验证导出的 IDF 的几何形状、表面标识符和结构以及对文件所做的任何修改。这对于验证由以下命令创建的矩形元素的修改特别有用 CONTAM3DExport ，包括必须修改为具有倾斜表面并向建筑物添加遮阳组件的屋顶。

  1. 在 contam.prj 文件和 IDF 文件中将时间步长以及开始日期和结束日期设置为相同。
  2. 如果需要，可以进行耦合模拟以利用重新启动文件。这可能有利于获得 CONTAM 模型内的初始气流和温度，但对于污染物汇和过滤器的加载可能是不利的。可以通过文本编辑器修改 contam.prj 的以下行，将重新启动标志设置为 1，并将日期设置为耦合模拟的第一天，从而使用重新启动文件进行模拟。在设置重启功能之前，不再需要通过 ContamW 运行仿真，但这仍然是一种有效的方法。修改 contam.prj 文件后，您必须替换压缩的 Contam.FMU 文件中现有的 PRJ 文件。
