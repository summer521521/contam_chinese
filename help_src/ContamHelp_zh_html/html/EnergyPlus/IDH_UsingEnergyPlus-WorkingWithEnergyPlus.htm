与 EnergyPlus 合作

从 CONTAM 3.2 开始，ContamX 可以与 EnergyPlus 结合执行组合气流和传热模拟 [ 多尔斯等人。 2016年 ]。这种耦合是使用 CONTAM 先前开发的进程间通信应用程序编程接口 (API) 来执行的。该 API 由一组套接字通信消息组成，可控制 ContamX 与外部进程之间的执行和数据交换。 先前已利用相同的套接字通信功能或桥接模式将 CONTAM 与 TRNSYS 耦合（ www.trnsys.com ）如记录在 与 TRNSYS 合作 部分。套接字通信 API 记录在 附录B 。 ContamX 和 EnergyPlus 之间的耦合是通过基于协同仿真规范版本 1.0 的功能模型接口 (FMI) 的协同仿真来完成的 莫迪萨2010 ]。 EnergyPlus 提供了控制模拟的主算法 [ 美国能源部 2013 , 努伊迪等人。 2014年 , 湿润2011 ]，并且 NIST 开发的模块 (ContamFMU.dll) 被合并到功能模型单元 (FMU) 中以供在执行期间使用。 因此，协同仿真是通过运行 EnergyPlus 以及配置为与 CONTAM 协同仿真的 IDF 文件来完成的，如本节所述。

NIST 开发了一个模块，可以合并到功能模型单元 (FMU) 中，以便在联合仿真期间使用。该模块ContamFMU可以在Windows下作为动态链接库（ContamFMU.dll）使用，也可以在Linux下作为共享对象（ContamFMU.so）使用。

EnergyPlus根据联合仿真描述文件建立自身与从FMU之间的动态数据交换。目前，EnergyPlus 和 ContamX 之间可以交换以下数据：

  • 从 EnergyPlus 到 ContamX
    • 房间温度
    • 房间湿度比 仅用于将 ContamX 质量流量转换为 EnergyPlus 体积流量
    • 通风系统风量 用于房间送风和回风流量
    • 室外气流分数 室外气流控制器
    • 排气扇风量
    • 能量回收风机气流速率
    • 室外环境数据 包括温度、 湿度比， 从 EnergyPlus 天气 (.epw) 文件获取的气压、风速和风向
    • 输出变量 可能是 EnergyPlus Output:Variable 或 EnergyManagementSystem:OutputVariable 对象
  • 从 ContamX 到 EnergyPlus
    • 房间渗透气流速率
    • 房间间气流速率 通过气流路径，即不通过管道
    • 控制值 它可以是通过 CONTAM 控制网络中的命名“分离”型控制节点公开的任何值，例如，由于污染物水平升高而要求通风气流的信号

The CONTAM3DExporter 该工具由 NIST 开发，用于协助 CONTAM 和 EnergyPlus 之间的耦合。该工具读取 CONTAM PRJ 文件并创建 EnergyPlus IDF 文件，其中包含与几何形状、渗透和房间间混合流相关的对象、包括室外空气混合器和 HVAC 气流终端的空气回路以及促进动态数据交换所需的 FMI 相关实体。下表提供了 CONTAM 和 EnergyPlus 实体之间的详细映射。表中具有固定宽度字体的项目，例如， 房间混合 ，表示属于 CONTAM 或 EnergyPlus 中对象的实体。斜体项目表示可通过 EnergyPlus 外部接口访问的 EnergyPlus 变量名称，例如， 房间平均气温 。注释列提供 CONTAM 和 EnergyPlus 实体之间关系的详细信息。
