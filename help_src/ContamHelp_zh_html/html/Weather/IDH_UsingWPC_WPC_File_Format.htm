WPC 文件格式

该文件提供连接到环境的每个流路的环境压力和污染物浓度值。

WPC 文件有两个版本 2.1 和 2.4。对于版本 2.1，压力必须以绝对压力的形式输入，单位为 Pa。风压将根据 WPC 文件中每个时间间隔提供的环境压力和密度以及每个流路的绝对压力和高程计算得出，如下所示：

Pwind = Ppath - Pambt + r •g•Zpath

 

对于 2.4 版本文件，WPC 文件中的压力将为风压本身，单位为 Pa。

污染物浓度必须以每千克空气中污染物千克数为单位提供。参见 浓度换算 在 污染物文件 的小节 处理物种和污染物 换算系数部分。

-----

WPC 文件的第一行用于标识文件的类型。对于 2.1 格式的 WPC 文件，它完全是：

WPC文件 ContamW 2.1

或者对于 2.4 格式的 WPC 文件，它正是：

WPC文件 ContamW 2.4

第二行是用户输入的文件的描述：

desc[] // 文件描述 (I1) {W};可能为空

后续行包含：

nPath // 流路数 (I2)

nCtm // 污染物数量（种类）(I2) [0 可能]

UsePres // 1 = 使用压力，0 = 不使用 (I2)

dtmin // 最小时间步长 (I2)（ContamX 未使用）

StartDate // WPC 数据的第一个日期 (mm/dd → IX)

EndDate // WPC 数据的最后日期 (mm/dd → IX)

I1 Name[1] // 污染物 1 的名称（I1 - 最多 16 个字符）

I1 Name[2] // 污染物 2 的名称 (I1)

...

Name[nCtm] // 污染物名称 NumCont (I1)

接下来的 NumPath 行描述了每个流路径：

number // 序列号 (I2)

X // X 坐标 (R4)

Y // X 坐标 (R4)

Z // Z 坐标 (R4)

map // 映射 [0 = 确定; 1 = 错误] (I2)

对于每天的 StartDate 到 EndDate：

对于每个日期的每个时间 00:00:00 到 24:00:00：

第一行：

日期 // 日期（月/日 → IX)

time // 一天中的时间 (hh:mm:ss → I4)

Pamb // 环境压力 [Pa] (R4)（如果使用压力）

dens // 空气密度 [kg/m3] (R4)（如果使用压力）

第二行：（如果使用压力）

P[1] // 路径[1] 处的压力 [Pa] (R4)

...

P[nPath] // 路径[nPath] 处的压力 [Pa](R4)

第三行（对于第一种污染物，如果 nCtm > 1):

C[1] // 路径[1] 处的浓度 [kg/kg 空气] (R4)

...

C[nPath] // 路径 [nPath] 处的浓度 [kg/kg 空气] (R4)

第四行（针对第二种污染物，如果需要）：

...

最后一行（污染物 nCtm ，如果需要）：

C[1] // 路径[1] 处的浓度 [kg/kg 空气] (R4)

...

C[nPath] // 路径 [nPath] 处的浓度 [kg/kg 空气] (R4)

 

文件描述不能以“开头 ! StartDate和EndDate用于验证文件数据是否涵盖了要模拟的整个时间段。StartDate不能晚于EndDate。

数据必须在 StartDate 的时间 00:00:00 开始，并在 EndDate 的 24:00:00 结束。 时间必须是连续的，但连续时间之间的差值不必是恒定的。
