PLD 文件格式

PLD（路径位置数据）文件将包含将建筑物连接到外部环境的任何流动路径的位置，以及用于指导 EWC 文件转换器行为的附加信息。该文件由 ContamW 创建，并存储在与项目文件相同的目录中，其名称与项目文件相同，但扩展名为 .pld 替换了 .prj 扩展名。它是根据需要创建的，用于将 WPC 文件信息与 CONTAM 项目文件中的信息进行比较。 （请参阅使用 WPC 文件）

该文件采用 ASCII 行分隔格式。

参考位置由 (Xref,Yref,Zref) 定义，表示 EWC 文件位置，相当于路径位置的 CONTAM 坐标系的原点。   用户必须在 EWC 文件和 PRJ 文件之间进行协调，以确定该位置的坐标值。

旋转角度围绕 Z（垂直）轴。   用户必须协调 EWC 和 PRJ 文件坐标之间的旋转方向   假设   没有绕 X 轴或 Y 轴旋转。

第 6 行的六个坐标定义了一个围绕该文件内的流路的边界框，可用于帮助 EWC 文件转换器在绘图过程中快速排除不需要的点（例如，距离建筑物太远的点）。   请注意，此边界框不一定包围整个建筑物。

模拟时间步长向 EWC 文件转换器提供减小 WPC 文件大小的建议，以便仅包含与模拟时间步长一致的时间步长（这可以减少所需的插值）。  

每个流路径由流路径类型和流路径ID值的组合唯一标识。

注释以感叹号 ('!') 开头，可以从任何行的开头开始（这样整行将被忽略），也可以在一行的所有字段之后开始（这样该行的其余部分将被忽略）。  

为了用户可读性，应尽可能对字段进行注释。

R4 数据类型的格式可以类似于 C scanf() 语句的转换规范来解释。

标题部分：

前三行：

EWC 文件名     //（I1完整路径）

WPC 文件名    //（I1完整路径）

描述[]    // 文件描述 (I1)

下一个标题行正是：

！外部参照     外部参照     伊雷夫     角度

下一行包含以米为单位的参考坐标：

外部参照     // (R4 %7.3f)

伊雷夫     // (R4 %7.3f)

兹雷夫     // (R4 %7.3f)

旋转   角度    // 真实格式%8.3f

下一行包含以度为单位的纬度和经度：

纬度     // (R4 %8.4f) 正北

经度    // (R4 %8.4f) 东部为正

下一个边界框标题行正是：

！最小     最大X值     伊敏     最大Y值      兹敏     最大Z值

下一行包含边界框坐标：

最小     // (R4 %7.3f)

最大X值     // (R4 %7.3f)

伊敏     // (R4 %7.3f)

最大Y值     // (R4 %7.3f)

兹敏     // (R4 %7.3f)

最大Z值     // (R4 %7.3f)

下一个时间步标题行正是：

！步骤   转变      开始    end

下一行包含时间步数据：

时间步长    // (I4) [s]

数据移位 //（时:分:秒 → I4) 转换为 WPC 时的 EWC 开始时间

开始   日期   //（毫米/日 → IX)

结束日期     //（毫米/日 → IX)

下一行包含：

风   按   旗帜     // (I4) 1：在 WPC 中包含压力，0：不包含

物种数量    //（I4）

物种图容差 // (R4 %7.3f) 摩尔质量容差

污染物定义部分：

污染物定义部分以标题行开头：

！姓名   重量

下一个 （物种数量） 每个物种的行包含：

名称[]      //（I1 - 最多 16 个字符）

摩尔质量   // (R4 %6.2f)

气流路径定义部分：

气流路径定义部分的第一行包含：

流路数量 // (I4)

路径图容差     // (R4 %7.3f) [m]

下一个标题行正是：

！编号#    X         Y       Z

下一个 (流路数) 每个气流路径的行包含：

路径号      //（九）

x 坐标     // (R4) [米]

y 坐标     // (R4) [米]

z 坐标    // (R4) [米]

该文件的最后一行正是：

-999
