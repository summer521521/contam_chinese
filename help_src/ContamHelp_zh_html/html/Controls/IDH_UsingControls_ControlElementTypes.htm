控制元件类型

有多种控制元素类型可以与未定义的控制节点关联。以下部分提供了这些控制元素的详细描述和属性。

传感器

根据与传感器关联的图标类型，可以创建两种类型的传感器： 房间或交界处传感器 and 路径或管道传感器 。传感器定义对话框将为您提供可用于正在创建的传感器类型的属性。

房间或交界处传感器

感知值： 选择您希望传感器作为输出提供的房间或交汇点的属性： 温度 , 污染物质量分数 , 表压 or 入住率 .

路径或管道传感器

感知值： 选择您希望传感器作为输出提供的路径或管道的属性： 流量 通过路径或 压降 穿过小路。

传感器输出： 传感器输出默认为 CONTAM 的基本单位，但您可以通过更改 增益 and 偏移量 提供的值。但是，“报告值”控制元素（如下所述）会自动提供从基本单位转换为项目或您检测到的特定污染物的默认单位的值。

输出 = (传感器挑战值   – 偏移）* 增益

增益 : 用于转换的乘数 桑斯 或挑战 价值 到 Out 值

偏移量： 用于转换的偏移值 感知值 到 O 输出 值

时间常数： 传感器响应时间常数，以秒为单位。 ContamX 将使用它来计算延迟因子。

延迟=e -Dt/t

传感器输出由转换后的值计算得出 传感器挑战值 对于当前时间步 t (Out)，前一个时间步 t- 的输出 D t ，以及 延迟 值如下：

输出（t）= Out + [输出(t- D t) – Out] * 延迟

地点： 这些参数仅适用于 质量分数传感器 与一维房间相关。选择是否希望传感器提供 房间平均值 一维房间的浓度或单个房间内的值 一维单元 一维房间。一维单元选项仅在使用 短时间步长 方法，否则传感器将提供基于房间平均值的值。

X、Y and 相对高程 (Z) 是所选传感器的坐标和单位。 X 和 Y 是绝对坐标，相对高程是相对于传感器所在水平面的。对于位于 1D 单元内的任何传感器都需要这些值 对流/扩散区 （见 一维房间数据 ）。如果传感器位于 1D 房间的特定单元内，则这些坐标将通过 ContamW 对照其所在 1D 房间的轴进行验证。当您选择以下任一选项时，将自动执行此验证 R 联合国模拟 or 创建一个Contam X 输入文件 命令来自 S 模拟 菜单。

说明： 用它来详细描述传感器。当您突出显示传感器图标时，此说明将显示在状态栏中。描述默认为“房间传感器”或“路径传感器”。

幻象控制

使用此控制元素可以引用项目中其他位置已存在的命名控制元素。当您需要来自位于 SketchPad 远程节点的控制信号时，或者当您正在实现多输入元素之一时，幻象节点非常有用。这在引用建筑物另一层的节点时非常有用，因为 ContamW 不允许您在建筑物级别之间绘制链接。这也可以用于将输入“级联”到多输入中 sum and 平均 当您想要对三个以上输入或远程命名输入进行求和或平均时进行控制。

时间表

使用此控制元素通过控制网络应用时间表。时间表可应用于气流路径、简单空气处理系统、简单空气处理系统的入口和出口以及源/汇。控制计划将覆盖定义为元素参数的计划。您可以选择现有计划或创建新计划（请参阅创建计划）。

常数

使用此控制元素可以定义一个简单的常量。该常数可用作其他控制元素的输入，例如 数学的 , 限制 and 开关 控制。

修饰符

使用此控制元素可通过提供增益和偏移参数来修改输入信号。输入信号将根据以下等式进行修改。

输出 = ( 输入 – 偏移）* 增益

Modifier 控件的参数为：

增益： 用于转换的乘数 输入 信号

偏移量： 用于转换的偏移值 输入 信号

迟滞

报告一个值

使用此控制元素将值报告到由 ContamW 创建的文件，用于记录每个的这些报告值的特定目的 列出时间步长 瞬态模拟（请参阅输出属性）。如果您执行瞬态仿真并实施 报告控制元素 , ContamW 将在与项目文件相同的目录中创建一个文件，并附加项目文件的名称和 .LOG 扩展名。例如，如果您的项目是 我的项目.PRJ , ContamW 将创建该文件 我的项目日志 。您还可以使用“报告”控制元素提供传感器数据的自动单位转换，从传感器的无量纲单位（例如 K 或 kg/kg）到与感测数据类型相关的工程单位（例如 ℃ 或 ppm(v)）。

报表元素的输出值将根据以下等式确定：

输出 = ( 输入 – 偏移) * 增益

报告控件的参数为：

增益： 用于转换的乘数 输入 信号

偏移量： 用于转换的偏移值 输入 信号

标题： 将出现在该控制元素的日志文件中的标头

单位： 将出现在该控制元素的日志文件标题下方的单位

如果 ContamW 在控制网络中发现尚未从无量纲形式转换的传感器“下行链路”，则报告控件还将提供默认转换值、增益、偏移和单位。

信号分割

使用它来分割信号，因此同一信号可以直接链接到控制网络中的多个节点。

逻辑性

使用这些控件提供逻辑操作，包括：

二进制 – 将输入信号转换为二进制 0 或 1，

NOT – 求反单个输入值并提供二进制输出 0 或 1，

AND – 对两个输入值执行 AND 运算并提供二进制输出 0 或 1。

输出 = 1  如果（ 输入1 > 0) 和 ( 输入2 > 0)

输出 = 0  否则

OR – 对两个输入值执行 OR 运算并提供二进制输出 0 或 1。

输出 = 1  如果（ 输入1 > 0) 或 ( 输入2 > 0)

输出 = 0  否则

XOR – 对两个输入值执行 XOR 运算并提供二进制输出 0 或 1。

输出 = 1  如果（ 输入1 > 0) 和 ( 输入2 <= 0)

输出 = 1  如果（ 输入2 > 0) 和 ( 输入1 <= 0)

输出 = 0  否则

数学

使用这些控件提供数学运算，包括：

绝对值 – 提供输入信号的绝对值

加、减、乘、除 – 对两个输入信号执行运算并提供结果作为输出。

请注意，操作顺序对于减法和除法很重要。

减去： 输出 = 输入1 – 输入2

划分： 输出 = 输入1 / 输入2

总和与平均值 – 求和或平均多个输入信号并提供结果作为输出信号。三个信号可以通过绘制在控制节点图标上的链接直接输入，更多的信号可以通过使用 幻影 节点。

经验值(x) – e 的 x 次方。

逻辑N(x) – 计算 x 的自然对数。

LOG10(x) – 计算 x 以 10 为底的对数。

战俘（x，y） – x 的 y 次方，其中 x 是 输入1 y 是 输入2 .

平方根 (x) – 计算 x 的平方根。

多项式(x) – 根据用户定义的多项式系数计算 x 的多项式。

正弦(x) – 计算 x 的正弦。

余弦(x) – 计算 x 的余弦。

正切值(x) – 计算 x 的正切。

MOD(x,y) – 计算 x 除以 y 的浮点余数，其中 x 是 输入1 y 是 输入2 .

最高限额(x) – 计算大于或等于 x 的最小整数。

楼层(x) – 计算小于或等于 x 的最大整数。

限位开关

限位开关 提供根据两个输入信号之间的差异将输出信号设置为开或关（0 或 1）的能力。

下限位开关

输出 = 1  如果（ 输入1 < 限制）      [极限= 输入2 ]

输出 = 0  否则。

上限位开关

输出 = 1  如果（ 输入1 > 限制）      [极限= 输入2 ]

输出 = 0  否则

频段开关包含控制参数， 带宽 ，提供一个 死区 用于开/关控制。

低频段开关

限制控制

限制控制 能够根据两个输入信号之间的差异将输出信号设置为 0.0 到 1.0 范围内的值。这些通常用于提供 错误信号 到比例控制器。

下限控制

输出 = 输入2 - 输入1

上限控制

输出 = 输入1 - 输入2

The 上限和下限控制 可用于准备一个错误信号 比例控制 .  如果输入信号到 上限控制 大于极限值，输出结果为正；   例如，如果污染物浓度太高，正信号可能会激活气流装置。   如果输入信号到 下限控制 低于极限值，输出结果为正；   例如，如果通过开口的气流太低，则正信号可以激活风扇以提供所需的流量。  

比例和比例-积分

比例控制 – 这是简单比例控制器的模型，其中输入信号是误差信号，通常是感测值减去通过使用其他控制函数获得的设定点值，例如的 持续控制 。输入信号乘以控制的比例常数参数， Kp 。输出信号限制为 0 到 1 之间的值，因此您应该相应地设置比例常数。然后可以使用其他控制元件根据需要修改输出信号。

输出 = 输入 × Kp

比例积分控制 – 这是一个简单的 P-I 控制器模型，其中输入信号是误差信号。输出信号限制为 0 到 1 之间的值，并由控制参数修改 Kp and Ki 根据以下等式。

输出 = 输出 *+Kp × ( 输入 - 输入 *) + 基 × ( 输入 + 输入 *)

哪里 输出 * 和 输入 * 是以下值 输出 and 输入 在前一个时间步。必须针对特定问题和时间步长调整 Kp 和 Ki 因子。这类似于[Stoecker and Stoecker 1989]的方程(17-37)和(18-3)中描述的“增量”P-I算法。

连续值文件和离散值文件（CVF/DVF）

这些控制元素允许您通过从文件获取输入值来实施常规计划。这允许您创建不受 12- 日程安排 CONTAM 的限制 周时间表 。这些是您根据中指定的格式创建的文本文件 CVF 格式 and DVF 格式 本文档的各部分。每次模拟只能使用每个文件之一，但该文件可能包含多个值列表。值列表被引用 值名称 ，它们是文件中的列标题。

CVF 和 DVF 文件不仅在格式上不同，而且在行为上也不同。在模拟期间，CONTAM 将在 CVF 值之间进行插值，但对于 DVF 值则不然，即，它们将保持不变，直到在文件中遇到新值。

从 CONTAM 版本 3.2 开始，现在可以在不使用关联的控制节点/网络的情况下使用这些文件，即，构建可能与计划关联的组件可以通过直接将它们与 值名称 CVF/DVF 文件的。现在，这在 ContamW 中每个组件的 Schedule 属性中都很明显。

值名称： 从标题列表中选择 CVF/DVF 文件中显示的一组值的名称。

文件名： 此字段显示 CVF/DVF 文件，其中包含控制节点在仿真期间将使用的数据。您必须使用 Data®Con 巨魔 ®C 连续值文件... and Data®Con 巨魔 ®D 离散值文件... 菜单项来选择在创建 CVF/DVF 控制节点之前要使用的文件。
