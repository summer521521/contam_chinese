连续值文件 (CVF) 格式

您可以使用 CVF   文件来实施时间表，这些时间表不限于 CONTAM 时间表等十二天类型。 CVF 值可通过具有 CVF 文件扩展名的连续值文件提供给 ContamX。 During simulation, ContamX will linearly interpolate between the values provided in the CVF   文件。您可以使用外部程序根据以下格式创建文本文件。

CVF 的第一行用于标识文件的类型。正是：

ContinuousValuesFile ContamW 2.1

第二行是用户输入的文件的描述：

描述[]          // 文件描述 (I1) {W};可能为空

下一行定义文件涵盖的时间段：

开始日期       // DEF 数据的第一个日期 (mm/dd ® IX)

结束日期         // DEF 数据的最后日期 (mm/dd ® IX)

下一部分指定控制节点名称：

_nbvfn          // CVF 节点名称的数量

接下来是由节点名称组成的 _nbvfn 行：

名称[]           // 节点名称 (I1)

文件的其余部分包含从 StartDate 到 EndDate 的每个日期和时间的所有节点的数据：

日期             // 日期（月/日 ® IX)

时间             // 一天中的时间 (hh:mm:ss ® I4)

值[1]         // 名称 [1] 的值 [?] (R4)

...

值[_nbvf]    // 名称 [_nbvf] 的值 [?] (R4)

节点数据必须在 StartDate 的时间 00:00:00 开始，并在 EndDate 的 24:00:00 结束。   时间必须是连续的，但连续时间之间的差值不必是恒定的。

文件描述不能以“开头 ! ”。的 开始日期 and 结束日期 用于验证文件数据是否覆盖了要模拟的整个时间段。的 开始日期 可能不会晚于 结束日期 。单行上的数据元素由制表符分隔。数据必须按时间顺序排列。文件值必须采用控制节点创建的信号所需的单位。

节点名称不得包含嵌入的空格。不在项目文件中的节点数据将被忽略。如果项目文件中的 CVF 节点名称未包含在 CVF 中，将导致致命错误。 ContamW 将通过检查节点名称来帮助用户。 ContamX 将在模拟开始之前执行名称检查。
