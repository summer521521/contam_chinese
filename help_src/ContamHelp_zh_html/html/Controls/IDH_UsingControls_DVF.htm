离散值文件 (DVF) 格式

您可以使用 DVF  files to implement schedules that are not limited to the twelve day types as are CONTAM schedules. DVF 值可通过具有 DVF 文件扩展名的离散值文件提供给 ContamX。顾名思义，ContamX 将 not 在 DVF 中提供的值之间进行插值   文件。您可以使用外部程序根据以下格式创建文本文件。

DVF 的第一行用于标识文件的类型。正是：

离散值文件 ContamW 2.1

第二行是用户输入的文件的描述：

描述[]          // 文件描述 (I1) {W};可能为空

下一行定义文件涵盖的时间段：

开始日期       // DEF 数据的第一个日期 (mm/dd ® IX)

结束日期         // DEF 数据的最后日期 (mm/dd ® IX)

下一部分指定控制节点名称：

_ndefn          // DVF 节点名称的数量

接下来是 _ndefn 行，其中包含：

名称[]          // 节点名称 (I1)

值           // 初始值[?] (R4)

每当节点值更改时，后续行都会显示数据：

日期            // 日期（月/日 ® IX)

时间            // 一天中的时间 (hh:mm:ss ® I4)

指数            // 索引到 name[]

值           // 新值 [?] (R4)

文件描述不能以“开头 ! ”。的 开始日期 and 结束日期 用于验证文件数据是否覆盖了要模拟的整个时间段。的 开始日期 可能不会晚于 结束日期 。单行上的数据元素由制表符分隔。数据必须按时间顺序排列。多个节点可能同时发生变化。文件值必须采用控制节点创建的信号所需的单位。

节点名称不得包含嵌入的空格。不在项目文件中的节点数据将被忽略。如果项目文件中的 DVF 节点名称未包含在 DVF 中，将导致致命错误。 ContamW 将通过检查节点名称来帮助用户。 ContamX 将在模拟开始之前执行名称检查。
