程序执行

为了激活 ContamX 的套接字通信功能，必须使用以下命令行结构运行程序。 Type98 和 ContamFMU.dll 都调用 污染x3 使用适当的命令行选项。

污染x3 < PRJ 文件名 > –b [地址:端口] [-w] [-f]

 

污染x3   程序可执行文件

< 项目 >  CONTAM 项目文件的名称

-b  在桥接模式下运行 ContamX（必需）

以下选项（如果提供）必须紧跟在“-b”之后 :

地址   将控制此 ContamX 实例（客户端）的进程（服务器）的 IP 地址

港口  套接字（即侦听套接字）的端口号，用于建立连接。监听套接字由服务器创建，例如 ContamFMU。然后，ContamX（客户端）创建另一个套接字，通过该套接字进行通信，即通过 connect() 函数。一旦服务器建立连接，即通过accept()函数，监听套接字就会关闭。然而，通信套接字保持打开状态。

-w ContamX 计算 w 工业压力 使用 PRJ 文件确定的传统方法（无、常量或变量）。应通过 ADJ_WTH_MSGTYPE 消息向 ContamX 提供可变压力流路的风速和风向。如果未使用此命令行选项，则必须通过 ADJ_WPC_MSGTYPE 消息提供风压和环境污染物。

-f contamx 套接字消息提供气流速率 体积 flow 单位 [米 3 /s]，而不是默认的 [kg/s]。

 

ContamX启动后，它将尝试与控制应用程序（即服务器）联系，并建立套接字连接。一旦建立连接，ContamX 将发送 _INFO_MSGTYPE 向服务器发送消息以提供有关项目文件内容的信息。这些项目信息消息将按数字顺序发送。然后，ContamX 将执行稳态初始化计算并发送所有 模拟结果 消息（ _UPDATE_MSGTYPE ）随后是 CX_READY_MSGTYPE 。此时，服务器可以开始通过桥接模式与 ContamX 交互 执行控制 and 项目修改 消息详述如下。 ContamX 将响应请求 _UPDATE_MSGTYPE 消息并通过发送来指示何时准备好接收更多消息 CX_READY_MSGTYPE 消息。

注意： 设置 _回声 PRJ 文件中的值大于 1，以便将桥消息信息写入 XLOG 文件。这可以提供对桥执行行为的深入了解。
