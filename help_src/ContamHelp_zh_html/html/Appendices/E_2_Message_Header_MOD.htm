消息头和消息 ID

消息头是每条消息的第一部分。它提供了建立要遵循的特定信息的一致方法。具体情况将取决于 类型 消息的字段。的 长度 字段包括标头的长度和随后的消息的特定部分。的 版本 提供字段是为了使利用 ContamX 的套接字通信功能的程序能够确保符合由于方法的修改/增强而发展的消息传递要求。

字符 模式[8]； // 用于验证标头“ACATMSG”的字符串。

int 长度； // 整个消息的长度（以字节为单位）

int 类型； // 消息类型（见下表。）

int 版本； // 当前消息版本 # = 10

 

消息类型定义

每个ID后面的值指示引入或修改消息的消息版本。消息版本必须与 ContamX 版本匹配，否则会出现错误。
