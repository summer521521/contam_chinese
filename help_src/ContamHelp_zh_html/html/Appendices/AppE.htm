附录 B - ContamX TCP/IP 套接字通信

ContamX 的套接字通信功能使另一个应用程序能够在 ContamX 运行时控制其执行。在此期间 桥 在执行模式下，ContamX 称为客户端，控制程序称为服务器。这两个程序使用 TCP/IP Socket 通信方法和一组预定义的消息相互通信。一般情况下，服务器必须由用户自行开发。本节提供了一组套接字通信消息，可在运行时交换 ContamX 相关信息，包括详细的气流结果、房间温度和控制数据。

ContamX 与 TRNSYS 和 EnergyPlus 结合的能力也已得到开发。 TRNSYS Type98 的开发是为了演示这些功能，并且 ContamFMU 动态链接库的开发是为了实现 EnergyPlus 和 ContamX 之间的协同仿真（请参阅 与 TRNSYS 合作 and 与 EnergyPlus 合作 ).

本节提供在套接字模式下执行 ContamX 的详细信息以及 ContamX 实现的预定义消息集的规范。这些消息属于从 ContamX 角度定义的以下类别：

  • 发送 项目信息
  • 发送/接收 执行控制
  • 接收 项目修改
  • 发送 模拟结果
