第 17 节：管道连接处

连接点数据由 jct_read( ) 函数读取并由 jct_save( ) 函数保存。数据存储在 JCT_DSC 结构中，该结构在 contam.h（针对 ContamW）和 simdat.h（针对 ContamX）中定义。

连接部分开始于：

_njct // 路口数量 (I2)

接下来是数据头注释行，然后是所有 _njct 连接点的数据。

对于每个连接点，数据行包括：

nr // 路口号 (I2);按从 1 到 _njct 的顺序

flags // 连接标志 – contam.h (U2) 中定义的位

jtype // 连接类型标志：0=jct，1=term，3=term w/ Wind {Contam 2.1}

pzn // 周围房间索引 (I2);转换为指针

dnr // 管道段编号 - 在读取期间使用 (I2)

pk // 动力学反应指数（I2）；转换为指针

ps // 调度索引 (I2);转换为指针

pc // 控制节点索引（I2）；转换为指针

pld // 级别索引 (I2);转换为指针

X // 环境终端的 X 坐标 [m] (R4) {Contam 2.1}

Y // 环境终端的 Y 坐标 [m] (R4) {Contam 2.1}

relHt // 相对于当前水平面的高度 [m] (R4)

T0 // 初始结温 [K] (R4)

P0 // 初始结压 [Pa] (R4)

icon // 用于表示路口 (U1) {W} 的图标

color // 图标颜色 (I2) {W} {Contam 2.4}

u_Ht // 高度单位 (I2) {W}

u_XY // X 和 Y 的单位 (I2) {W}

u_T // 温度单位 (I2) {W}

u_dP // 压力差单位 (I2) {W}

vf_type // 0=无值文件，1=使用 cvf，2=使用 dvf (I2) {CONTAM 3.2}

vf_node_name[NAMELEN] // 值文件节点名称 (I1) {CONTAM 3.2}

如果j类型 > 0 然后该行继续：

"T:" // 字符串表示终端数据遵循{Contam 2.3}

pf // 过滤器索引 (I2);转换为指针 {Contam 2.3}

pw //风系数指数(I2);转换为指针

wPset //恒定风压[Pa] (pw==NULL) (R4)

wPmod // 风速（？）修饰符（pw！= NULL）（R4）

wazm // 墙壁方位角 (pw!=NULL) (R4)

bal // 余额终端标志 (I2) {Contam 2.3}

Ad // 连接到终端的管道面积 [m^2] (R4) {Contam 2.3}

Af // 航站楼可用面积 [m^2] (R4) {Contam 2.3}

Fdes // 设计流量[kg/s] (R4) {Contam 2.3}

Ct //终端损耗系数（R4）{Contam 2.3}

Cb //平衡损失系数（R4）{Contam 2.3}

Cbmax // 所需的最大 Cb (R4) {Contam 2.3}

u_A  // 面积单位 (I2) {Contam 2.3}

u_F // 质量流量单位 (I2) {Contam 2.3}

ddir // 终端管道方向 - 显示风压 (U1) {W}

fdir // 正流向 - 显示流向 (U1) {W}

连接部分终止于：

-999 // 用于检查上述数据是否有读取错误
