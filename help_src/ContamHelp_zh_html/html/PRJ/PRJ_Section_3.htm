第 3 部分：级别和图标数据

用于重新创建 SketchPad 的数据存储在该部分中。数据由level_read()函数读取并由level_save()函数保存。关卡数据存储在 LEV_DATA 结构中，该结构在 contam.h（对于 ContamW）和 simdat.h（对于 ContamX）中定义。

图标数据存储在 ContamW 中的 ICON_DAT 结构中，并且不保存在 ContamX 中。

级别/图标部分开头为：

nlev // 层数 ( IX)

接下来是数据标题注释行，然后是所有 nlev 级别的数据。

每个级别都有一条数据线，其中包括：

nr // 级别号 ( IX )，按从 1 到 nlev 的顺序

refht // 参考标高 [m] (R4)

delht // 到下一级的海拔增量 [m] (R4) {W}

nicon // 此级别上的图标数量 (IX)

u_rfht // 参考高程单位 (I2) {W}

u_dlht // 增量高程单位 (I2) {W}

name[] // 关卡名称 (I1)

该行后面是注释行，然后是 nicon 图标的数据。

每个图标都有一条数据线，其中包括：

icon // 图标类型 – 请参阅 contam.h 中的“特殊符号” (I2) {W}

row // SketchPad 上的行位置 (I2) {W}

col // SketchPad 上的列位置 (I2) {W}

nr // 房间、路径、管道等，编号 (I2) {W}

该部分以以下内容结束：

-999 // 用于检查上述数据（I2）中是否存在读取错误
