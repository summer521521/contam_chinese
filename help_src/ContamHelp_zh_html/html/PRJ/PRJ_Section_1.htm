第 1 部分：项目、天气、模拟和输出控制

项目文件的初始部分由run_read()函数读取并由run_save()函数保存。

项目文件的第一行标识源程序：

_string[] // 程序名称 "ContamW" (I1)

_string[] // 程序版本“3.4”(I1)

_回声 // > 0 回显输入文件， > 1 将桥接消息回显到日志文件 (I2)

第二行是用户定义的项目描述：

_prjdesc[] // （未使用）项目描述 (I1) {W}

下一行设置以下杂项值：

_skheight // SketchPad 总高度 [单元格] (I2) {W}

_skwidth // SketchPad 总宽度 [单元格] (I2) {W}

_def_units // 默认单位：0 = SI，1 = US (I2) {W}

_def_flows // 默认流量：0 = 质量，1 = 体积 (I2) {W}

_def_T // 房间的默认温度 [K] (R4) {W}

_udefT // 温度单位 (I2) {W}

_rel_N // 与正北的角度（-180 到 180）[度] (R4) {W}

_wind_H // 参考风速的海拔高度 [m] (R4) {W}

_uwH // _wind_H (I2) {W} 的单位

_wind_Ao // 局部地形常量 (R4) {W}

_wind_a // 速度剖面指数 (R4) {W}

下一行提供 SketchPad 伪几何信息：{CONTAM 3.1}

scale // 单元格缩放因子 (R4)

uScale // 缩放因子的单位：0 = m, 1 = ft (I2)

orgRow // 原点行 (I2)

orgCol // 起始列 (I2)

invYaxis // 0 = 原点上方为正值，1 = 原点上方为负值 (I2)

showGeom // 0/1 = 在状态栏中显示伪几何信息 (I2)

下一行定义 ContamX 中稳态模拟的天气 (WTHDAT)：

Tambt // 环境温度 [K] (R4)

barpres // 大气压力 [Pa] 未根据海平面进行校正 (R4)

Windspd // 风速 [米/秒] (R4)

Winddir // 风向: 0 = N, 90 = E, 180 = S, ...; (R4)

relhum // 相对湿度：0.0 至 1.0 (R4)

daytyp // 日期类型 (1-12) (I2)

uTa // Tambt (I2) {W} 的单位

ubP // barpres (I2) {W} 的单位

uws // Windspd (I2) {W} 的单位

uwd // Winddir (I2) {W} 的单位

下一行定义 ContamW 中风压测试的天气数据 (WTHDAT)：

Tambt // 环境温度 [K] (R4) {W}

barpres // 大气压力 [Pa] 未根据海平面进行校正 (R4) {W}

Windspd // 风速 [米/秒] (R4) {W}

Winddir // 风向: 0 = N, 90 = E, 180 = S, ...; (R4) {W}

relhum // 相对湿度：0.0 至 1.0 (R4) {W}

daytyp // 日期类型 (1-12) (I2) {W}

uTa // Tambt (I2) {W} 的单位

ubP // barpres (I2) {W} 的单位

uws // Windspd (I2) {W} 的单位

uwd // Winddir (I2) {W} 的单位

接下来的四行分别定义天气、污染物和控制文件：

_WTHpath[_MAX_PATH] // 天气文件全名 (I1)

_CTMpath[_MAX_PATH] // 污染物文件全名 (I1)

_CVFpath[_MAX_PATH] // 连续值文件的全名 (I1)

_DVFpath[_MAX_PATH] // 离散值文件的全名 (I1)

以下行定义用于创建 WPC 文件的路径位置数据 (PLDDAT)：

三行文件路径和 WPC 描述：

WPCfile[_MAX_PATH] // WPC 文件的全名 (I1)

EWCfile[_MAX_PATH] // EWC数据源文件全名(I1) {W}

WPCdesc[] // WPC 描述 (I1) {W}

接下来是描述参考坐标的一行：  

X0 // EWC 坐标中 ContamW 原点的 X 值 [m] (R4) {W}

Y0 // EWC 坐标中 ContamW 原点的 Y 值 [m] (R4) {W}

Z0 // EWC 坐标中 ContamW 原点的 Z 值 [m] (R4) {W}

angle // ContamW 相对于 EWC 坐标 (R4) {W} 的旋转

u_XYZ // 坐标单位 (I2) {W}

然后是一些 EWC 到 WPC 转换参数：

epsPath // 匹配路径位置的容差 [-] (R4) {W}

epsSpcs // 匹配物种的耐受性 [-] (R4) {W}

tShift // EWC 数据的时间平移 {W} [s] (hh:mm:ss ® 九) {W}

dStart // WPC 数据开始日期 (I2) {W}

dEnd // WPC 数据结束日期 (I2) {W}

_useWPCwp // 如果为 true，则使用 WPC 文件风压 (I2)

_useWPCmf // 如果为 true，则使用 WPC 文件质量分数 (I2)

wpctrig // 0/1 = 通过 WPC 数据触发气流计算 (I2) {CONTAM 3.1}

下一行定义位置 (LOCDAT)（供将来热模拟使用）：

latd // 纬度（度：北+，南-）（R4）

lgtd // 经度（度：东+，西-）（R4）

Tznr // 时区（格林威治 = 0，东部 = -5 等）(R4)

altd // 海拔高度 [m] (R4)

Tgrnd // 地面温度 [K] (R4)

utg // 地面温度单位 (I2)

u_a // 高程单位 (I2)

其余数据存储在运行控制（RCDAT）结构中。在 ContamX 中，某些值在使用之前可能会转移到其他变量。接下来的两行控制气流模拟 - 首先是非线性部分：

sim_af // 气流模拟：0 = 稳定，1 = 动态 (I2)

afcalc // 非线性方程的 N-R 方法：0 = SUR, 1 = STR (I2)

afmaxi // N-R迭代的最大次数(I2)

afrcnvg //相对气流收敛因子（R4）

afacnvg //绝对气流收敛因子[1/s] (R4)

afrelax // 流动欠松弛系数（对于 SUR）(R4)

uac2 // afacnvg (I2) 的单位

普雷斯    // 压力测试压力(R4) {Contam 2.4}

普雷斯    // Pres (I2) 的单位 {Contam 2.4}

然后是线性部分：

afslae // 线性方程方法：0 = SKY，1 = PCG (I2)

afrseq // 如果为 true，则对线性方程重新排序 (I2)

aflmaxi // 最大迭代次数 (PCG) (I2)

aflcnvg // (PCG) (R4) 的相对收敛因子

aflinit // 如果为 true，则进行线性气流初始化 (I2)

Tadj // 如果为 true，则使用温度调节 (I2)

接下来的四行控制用于污染物计算的线性求解器：

首先进行循环模拟：

sim_mf // 质量分数（污染物）模拟：

// 0 = 无，1 = 稳定，2 = 瞬态，3 = 循环 (I2)  

ccmaxi // 模拟：最大循环迭代次数（I2）

ccrcnvg // 相对收敛因子 (R4)

ccacnvg // 绝对收敛因子 [kg/kg] (R4)

ccrelax //（未使用）过松弛系数（R4）

uccc // cccnvg (I2) 的单位

那么对于非痕量污染物：

mfnmthd // 模拟：0 = SKY、1 = BCG、2 = SOR、3 = LU (I2)

mfnrseq // 如果为 true，则对线性方程重新排序 (I2)

mfnmaxi // 最大迭代次数 (I2)

mfnrcnvg // 所需的相对收敛 (R4)

mfnacnvg // 所需的绝对收敛 (R4)

mfnrelax //松弛系数(R4)

mfngamma //梯形积分因子（R4）

uccn // mfnacnvg (I2) 的单位

那么对于痕量污染物：

mftmthd // 0 = 天空，1 = BCG，2 = SOR，3 = LU (I2)

mftrseq // 如果为 true，则对线性方程重新排序 (I2)

mftmaxi // 最大迭代次数 (I2)

mftrcnvg // 所需的相对收敛 (R4)

mftacnvg // 所需的绝对收敛 (R4)

mftrelax //松弛系数(R4)

mftgamma //梯形积分因子（R4）

ucct // mftacnvg (I2) 的单位

然后对于 CVODE   污染物求解器：{CONTAM 3.1}
