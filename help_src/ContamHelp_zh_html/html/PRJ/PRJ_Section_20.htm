第 20 节：源/汇

污染物源/汇由 css_read() 函数读取并由 css_save() 函数保存。数据存储在 CSS_DSC 结构中，该结构在 contam.h（对于 ContamW）和 simdat.h（对于 ContamX）中定义。

源/汇部分开始于：

_ncss // 源/汇数量 ( IX)

接下来是数据头注释行，然后是所有 _ncss 源/汇的数据。

对于每个源/汇，数据线包括：

nr // 源/汇编号 ( IX ）；按从 1 到 _ncss 的顺序

pz // 房间索引 ( IX ）；转换为指针

pe // 源/宿元素索引 ( IX ）；转换为指针

ps // 调度索引 ( IX ）；转换为指针

pc // 控制节点索引 ( IX ）；转换为指针

mult // 乘数 (R4)

CC0 //初始质量分数（某些类型）[kg/kg]（R4）

Xmin // 源/汇位置的最小 x 坐标 [m] (R4)

Ymin //源/汇位置的最小 y 坐标 [m] (R4)

Hmin // 相对于水平面的最小 z 坐标 [m] (R4)

Xmax // 源/汇位置的最大 x 坐标 [m] (R4)

Ymax //源/汇位置的最大 y 坐标 [m] (R4)

Hmax // 相对于水平面的最大 z 坐标 [m] (R4)

color // 图标颜色 (I2) {W} {CONTAM 3.3}

u_XYZ // 位置坐标单位 (I2) {W}

vf_type // 0=无值文件，1=使用 cvf，2=使用 dvf (I2) {CONTAM 3.2}

vf_node_name[NAMELEN] // 值文件节点名称 (I1) {CONTAM 3.2}

cfd // cfd 路径 (0=否，1=是) (I2) {CONTAM 3.0}

如果 cfd = 1，则该行继续包含：{CONTAM 3.0}

name[] // cfd 路径 ID (I1)

源/汇部分终止于：

-999 // 用于检查上述数据是否有读取错误
