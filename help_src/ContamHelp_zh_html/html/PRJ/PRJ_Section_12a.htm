第 12a 节：控制超元件

控制超元件数据由 ctrlse_read( ) 函数读取并由 ctrlse_save( ) 函数保存在 ContamW 中。数据存储在 contam.h 中为 ContamW 定义的 CTRLSE_DAT 结构中。 ContamX 不会读入超元件数据，ContamX 只会读入从控制超元件实例化的节点，如以下第 12b 节所述。

超元件部分开头为：

_nselmt // 超元件的数量 (IX)

接下来是所有 _nselmt 超元件的数据。

对于每个超元件，第一数据行包括：

nr // 元素编号 (IX);按从 1 到 _nselemt 的顺序

旗帜    // 如果（标志 & CF_DEF ) 超元件已定义 (U2)

    // 如果（标志 & CF_SNS ) 超元件是传感器

in // 输入节点号 (IX)

out // 输出节点号（IX）

nn // 子节点数量（IX）

ni // 草图图标的数量 (IX)

name[] // 元素名称 (I1)

第二行有：  

desc[] // 元素描述（I1）可以为空

接下来的部分包含超元件的子节点数据。这与项目文件的“控制节点：”部分（参见下面的第 12b 节）非常相似，并且在第 12b 节中为从超元件实例化的每个超级节点重复。然而，实例化的超级节点的实际参数可能与超元件子节点的参数不同。

接下来是包含超元件草图的 SketchPad 图标参数的数据行，并将遵循“ 级别加上图标数据： ”部分，但仅适用于表示超元件子节点图标的单个“级别”数据（请参阅第 3 节）。

超元件部分结束于：

-999
