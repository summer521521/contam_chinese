第 8a 节：过滤元件

滤波器元件数据由flte_read()函数读取并由flte_save()函数保存。数据存储在 FLT_ELT 结构中，这些结构在 contam.h（对于 ContamW）和 simdat.h（对于 ContamX）中定义。

过滤元件部分开始于：

_nflte // 过滤器元素的数量 ( IX)

接下来是所有数据 _nflte 过滤器。

对于每个元素，第一个数据行包括：

nr // 元素编号 ( IX ）；按从 1 到 _nflte 的顺序

ftype // 元素数据类型（字符串？I2）

// 类型名称按 ftype 顺序存储在 _flte_names 中。  

area // 过滤器的面面积 [m] (R4)

深度 // 过滤器深度 [m] (R4)

dens // 过滤材料的密度 [kg/m^3] (R4)

// 注：面积 * 深度 * 密度 = 过滤器质量 [kg]  

ual // 面积和深度单位 (I2) {W}

ud // 密度单位 (I2) {W}

name[] // 元素名称 (I1) {W}

第二行有：

desc[] // 过滤器描述 (I1) {W} 可以为空

接下来是取决于元素数据类型的数据行。

接下来的介绍行给出了类型编号、定义为该编号的程序宏、来自 _flte_names 的类型名称以及保存数据的数据结构。数据结构在 ContamW 中的 celmts.h 和 ContamX 中的 selmts.h 中定义。只要 _flte_names 和相关数组中的顺序反映了该顺序，类型号就可以更改。

 

元素类型 0 [FL_CEF]“cef”存储在结构 CEF_DAT 中。

恒效过滤器数据包括：

nspcs // 物种数量 ( IX)

接下来是 国家政策委员会 过滤器效率数据行：

spcs[] // 物种名称 (I1)

eff // 过滤器效率 [-] (R4)

元素类型 1 [FL_PF0]“pf0”存储在结构 PF0_DAT 中。

简单粒子滤波器数据包括：

npts // 数据点数 (I2)

usz // 粒径单位 (I2) {W}

接下来是 不扩散条约 过滤器效率数据行：

size // 粒径 [m] (R4)

eff // 过滤器效率 [-](R4)

元素类型 2 [FL_GF0]“gf0”存储在结构 GF0_DAT 中。

简单的气体过滤器数据包括：

nspcs // 物种数 (IX)

接下来是 国家政策委员会 部分包括：

spcs[] // 物种名称 (I1)

bthru[] // 突破 eff (R4) {Contam 2.4}

npts // 数据点数 (I2)

其中每一个后面都跟着 不扩散条约 过滤器效率数据行：

load // 相对总负载 [-] (R4)

eff // 该物种的过滤效率[-] (R4)

元素类型 3 [FL_SPF]“spf”存储在结构 SPF_DAT 中。

超级过滤数据包括：

恩瑟尔姆特   // 子元素个数(I2)

然后是一行数据，其中包含按流顺序排列的每个子元素的编号列表：

埃尔姆特[]   // 子元素编号 (IX) – 转换为指针

过滤器元件部分终止于：

-999 // 用于检查上述数据是否有读取错误
