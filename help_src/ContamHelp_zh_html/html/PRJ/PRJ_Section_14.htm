第 14 节：房间

房间数据由zone_read()函数读取并由zone_save()函数保存。数据存储在 ZONE_DSC 结构中，这些结构在 contam.h（对于 ContamW）和 simdat.h（对于 ContamX）中定义。

房间部分开始于：

_nzone // 房间数量 ( IX)

接下来是数据头注释行，然后是所有 _nzone 房间的数据。

对于每个房间，数据行包括：

nr // 防区编号 ( IX ）；按从 1 到 _nzone 的顺序

flags // 房间标志 – contam.h (U2) 中定义的位

ps // 周日程索引 ( IX ）；转换为指针

pc // 控制节点索引 ( IX ）；转换为指针

pk // 动力学反应指数 ( IX ）；转换为指针

pl // 建筑等级索引 ( IX ）；转换为指针

relHt // 房间高度 [m] (R4)

Vol // 房间体积 [m^3] (R4)

T0 // 初始房间温度 [K] (R4)

P0 // 初始房间压力 [Pa] (R4)

name[] // 房间名称 (I1) {W}

color // 房间填充颜色 (I2) {W} {Contam 2.4}

u_Ht // 高度单位 (I2) {W}

u_V // 体积单位 (I2) {W}

u_T // 温度单位 (I2) {W}

u_P // 压力单位 (I2) {W}

cdaxis // 转换/差异轴 – (0=无 cd, 1-4 = > cd轴方向）（I2）

vf_type // 0=无值文件，1=使用 cvf，2=使用 dvf (I2) {CONTAM 3.2}

vf_node_name[NAMELEN] // 值文件节点名称 (I1) {CONTAM 3.2}

cfd // cfd 房间 (0=否，1=是) (I2) {CONTAM 3.0}

如果 cfd = 1，则该行继续包含：{CONTAM 3.0}

name[] // cfd 房间 ID (I1)

如果cdaxis > 0 然后该行继续包括：

"1D:" // 后面是表示一维数据的字符串

X1        // cdaxis一端的X坐标

Y1        // 是      "            "

H1        // 相对高度     "

X2        // cdaxis另一端的X坐标

Y2        // 是      "            "

H2        // 相对高度     "

celldx // c/d 单元的长度 [m] (R4)

axisD // 轴向扩散系数[m^2/s] (R4)

u_aD // 轴向扩散单位 (I2)

u_L // c/d 轴限制单位 (I2)

房间部分终止于：

-999 // 用于检查上述数据是否有读取错误
